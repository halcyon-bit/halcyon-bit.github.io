三、链式比较

Python 支持链式比较，如 3 < 4 < 5 这种就叫链式比较，相当于 3 < 4 and 4 < 5。

四、条件

def test(score=80):
    if 0 <= score < 60:
        level = 'D'
    elif 60 <= score < 80:
        level = 'C'
    elif 80 <= score < 90:
        level = 'B'
    elif score >= 90:
        level = 'A'
    else:
        level = 'E'
    return level

test(50)

# 条件成立时执行的语句 if condition else 条件不成立时执行的语句
score = 50
level = 'D' if 0 <= score < 60 else 'C'

六、元组

所谓的元组无法修改值的本质是不能修改当前这个值的引用。

x = (1, 2, 3, [4])

当前元组中的第一个元素在内存中已经保存了，元组中的索引指向了这个元素。因为当前的元组中嵌套了列表，也就是说索引所指向的这个值是一个列表类型，所以可以对当前这个列表中的元素进行修改，因为修改完之后还是列表类型。

九、函数参数

调用函数时可以使用以下参数类型：

  1. 位置参数
  2. 默认参数
  3. 可变参数（*args），接收的是一个 tuple
  4. 关键字参数（*kwargs），接收的是一个 dict

这 4 种参数可以组合使用。注意定义参数的顺序必须是位置参数、默认参数、可变参数、命名关键字参数、关键字参数。

def test(name, age, city='北京', *args, **kwargs):
    print(f'name={name}, age={age}, city={city}, args={args}, kwargs={kwargs}')

year = [2017,2018]
detail = {'sex':'man','interset':'coding'}
test('XXX',20)
test('XXX',20,'中国',*year,**detail)

# 输出：
# name=XXX, age=20, city=北京, args=(), kwargs={}
# name=XXX, age=20, city=中国, args=(2017, 2018), kwargs={'sex': 'man', 'interset': 'coding'}

class MyException(Exception):
    def __init__(self):
        pass

    def __str__(self):
        return 'this is my exception'

def test():
    try:
        raise MyException()
    except MyException as e:
        print("my exception is: ", e)

test()

十一、全局变量

函数中对于全局变量的修改：如果全局变量是不可变对象，修改时需要使用 global 声明；如果是可变对象，则不需要 global 声明。

num = 100
num_list = [100, 90]

def test_1():
    global num
    num = 90

def test_2():
    num_list.append(80)

print(num)
print(num_list)

test_1()
test_2()

print(num)
print(num_list)

	return ' '.join([hex(ord(c)).replace('0x', '') for c in s])

	return ''.join([chr(i) for i in [int(b, 16) for b in s.split(' ')]])

2. 线程

Python 代码的执行由 Python 虚拟机（解释器主循环）控制。Python 在设计之初就考虑到在主循环中只能有一个线程执行，虽然 Python 解释器中可以“运行”多个线程，但是在任意时刻只有一个线程在解释器中运行。

Python 虚拟机的访问由全局解释器锁（Global Interpreter Lock, GIL）控制，这个锁能保证同一时刻只有一个线程运行。

在多线程环境中，Python 虚拟机按以下方式执行：

  1. 设置 GIL
  2. 切换到一个线程运行
  3. 运行指定数量的字节码指令或线程主动让出控制（可以调用 time.sleep(0)）
  4. 把线程设置为睡眠状态
  5. 解锁GIL
  6. 再次重复以上所有步骤

在调用外部代码（如 C/C++ 扩展函数）时，GIL 将被锁定，直到这个函数结束为止（由于在此期间没有运行 Python 的字节码，因此不会做线程切换），编写扩展的程序员可以主动解锁 GIL。
