new 和 delete

cpp
后端
发布日期:   2020-08-10
文章字数:   1.9k
阅读时长:   7 分

new 和 delete 是操作符，不能被重载，内部实际调用对应的 operator 函数。

operator new 和 operator delete 函数有两个重载版本：

void* operator new(size_t);         // allocate an object
void* operator new[](size_t);       // allocate an array
void* operator delete(void*);       // free an object
void* operator delete[](void*);     // free an array

一、new

new 操作针对数据类型的处理，分为两种情况：

1. 简单数据类型

包括基本数据类型和不需要构造函数的类型。

int* p = new int;

汇编代码如下：

operator new 的源码如下：

void* __CRTDECL operator new(size_t size) _THROW1(_STD bad_alloc)
{
    void *p;
    while ((p = malloc(size)) == 0) {
        if (_callnewh(size) == 0) {
            _THROW_NCEE(_XSTD bad_alloc);
        }
    }
    return (p);
}

调用 malloc 失败后会调用 _callnewh，如果返回 0 则抛出 bad_alloc 异常；返回非零则继续申请内存。

这个 _callnewh 是一个 new handler，通俗来讲就是 new 失败的时候调用的回调函数。可以通过 _set_new_handler 来设置。

#include <new.h>
#include <iostream>

int myNewHandler(size_t size)
{
    std::cout << "Allocation failed. Try again\n";
    return 1;  // continue to allocate
    // return 0;  // stop allocating, throw bad_alloc
}

int main()
{
    // Set the failure handler for new to be myNewHandler.
    _set_new_handler(myNewHandler);
    while (1) {
        int* p = new int[10000000];
    }
    return 0;
}

在 new 基本数据类型的时候还可以指定初始化值：

int* p = new int(4);

总结

  • 简单类型直接调用 operator new 分配内存；
  • 可以通过 new handler 来处理 new 失败的情况；
  • new 分配失败的时候不像 malloc 那样返回 NULL，而是直接抛出异常。要判断是否分配成功应该用异常捕获机制。

2. 复杂数据类型

需要由构造函数初始化的对象。

class Object
{
public:
    Object() 
        : val_(1)
    {}

    ~Object()
    {}

private:
    int val_;
};

int main()
{
    Object* p = new Object();
}

总结

先调用 operator new，然后在分配的内存上调用构造函数。

二、delete

1. 简单数据类型

包括基本数据类型和不需要析构函数的类型。

int* p = new int(1);
delete p;

汇编码如下：

operator delete 的源码如下：

void operator delete(void* p)
{
    RTCCALLBACK(_RTC_Free_hook, (p, 0));
    free(p);
}

RTCCALLBACK 默认是空的宏定义，所以这个函数默认情况下就是简单的调用 free 函数。

总结

简单数据类型默认只是调用 free 函数。

2. 复杂数据类型

需要由析构函数销毁对象。

class Object
{
public:
    Object() 
        : val_(1)
    {}

    ~Object() 
    { 
        std::cout << "destroy object" << std::endl;
    }

private:
    int val_;
};

int main()
{
    Object* p = new Object;
    delete p;
    return 0;
}

总结

复杂数据类型先调用析构函数再调用 operator delete。

三、new 数组

1. 简单数据类型

包括基本数据类型和不需要构造函数的类型。

new[] 调用的是 operator new[]，计算出数组总大小之后调用 operator new。可以通过 () 初始化数组为零值。

char* p = new char[32]();
// 等价于
char* p = new char[32];
memset(p, 0, 32);

总结

针对简单类型，new[] 计算好大小后调用 operator new。

2. 复杂数据类型

需要由构造函数初始化对象。

#include <iostream>

class Object
{
public:
    Object() 
        : val_(1)
    {}

    ~Object() 
    { 
        std::cout << "destroy object" << std::endl;
    }

private:
    int val_;
};

int main()
{
    Object* p = new Object[3];
    return 0;
}

new[] 先调用 operator new[] 分配内存，然后在 p 的前四个字节写入数组大小，最后调用三次构造函数。

实际分配的内存块如下：

数组长度是对象析构时会使用的值。

class Object
{
public:
    Object() 
        : val_(1)
    {}

    ~Object() 
    { 
        std::cout << "destroy object" << std::endl;
    }

private:
    int val_;
};

class MyObject : public Object
{
public:
    ~MyObject()
    { 
        std::cout << "destroy MyObject" << std::endl; 
    }

private:
    int foo_;
};

int main()
{
    Object *p = new MyObject[3];
    delete[] p;
    return 0;
}

释放内存之前会调用每个对象的析构函数，但是编译器并不知道 p 实际所指对象的大小。如果没有储存数组大小，编译器如何知道该把 p 所指的内存分为几次来调用析构函数呢？

总结

针对复杂类型，new[] 会额外存储数组大小。

四、delete 数组

1. 简单数据类型

包括基本数据类型和不需要析构函数的类型。delete 和 delete[] 效果一样：

int* pNum = new int[32];
delete pNum;  // 等价于 delete[] pNum;

char* pch = new char[32];
delete[] pch;  // 等价于 delete pch;

运行后不会有什么问题，内存也能被释放。operator delete[] 就是简单的调用 operator delete。

注意：虽然 delete 和 delete[] 等同，但最好还是 new 与 delete 配对使用。

总结

针对简单类型，delete 和 delete[] 等同。

2. 复杂数据类型

需要由析构函数销毁对象，释放内存之前会先调用每个对象的析构函数。

这时 new[] 分配的内存只能由 delete[] 释放，如果由 delete 释放会崩溃。

假设指针 p 指向 new[] 分配的内存。因为要 4 字节存储数组大小，实际分配的内存地址为 [p-4]，系统记录的也是这个地址。delete[] 实际释放的就是 p-4 指向的内存。而 delete 会直接释放 p 指向的内存，这个内存根本没有被系统记录，所以会崩溃。

总结

针对复杂类型，new[] 出来的内存只能由 delete[] 释放。

五、malloc() vs new

malloc() new
库函数 操作符
申请失败时，返回 NULL 申请失败时，抛出 bad_alloc 异常
返回 void* 返回准确的数据类型

允许使用 realloc() 更改缓冲区的大小 不允许

六、placement new/delete

placement new 是 C++ 中的一种变体 new 运算符。

new (address) (type) initializer

普通的 new 运算符做两件事：（1）分配内存、（2）在分配的内存中构造一个对象。

而 placement new 允许将以上两件事分开。在 placement new 中，可以传递一个预分配的内存并在传递的内存中构造一个对象。

#include <iostream>
int main()
{
    int x = 10;
    std::cout << "x = " << x << std::endl;
    std::cout << "x addr: " << &x << std::endl;

    std::cout << "call placement new on x (new (&x) int(100);)" << std::endl;
    int* res = new(&x)int(100);

    std::cout << "x = " << x << std::endl;
    std::cout << "res = " << *res << std::endl;
    std::cout << "res addr: " << res << std::endl;
    return 0;
}

1. placement new 使用时机

因为它允许在已经分配的内存上构造一个对象，所以可能存在需要多次重新构造对象的情况，因此在这些情况下 placement new 运算符可能更有效。

#include <iostream>
class String
{
public:
    String(size_t len)
    {
        len_ = len;
    }
    ~String()
    {
        len_ = 0;
    }

public:
    size_t len_;
    char str_[];
};

int main()
{
    const char* str = "hello world";

    // 分配所需内存
    void* mem = ::operator new(sizeof(String) + strlen(str) + 1);

    // 在现有内存块上构造一个 String 对象
    String* s = new(mem)String(strlen(str));
    memcpy(s->str_, str, s->len_);
    // ...
    // 当不使用 String 时，在不释放内存的情况下销毁 String 对象
    s->~String();
    
    // 需要新的 String 时，可以继续在 mem 上构造
    // ...
    
    // 当不需要使用 mem 内存时，释放内存
    ::operator delete(mem);
    return 0;
}

2. 删除 placement new 分配的内存

对于类对象，需要显示调用类的析构函数。不可以使用 delete 操作符。

// 在现有内存块上构造一个 String 对象
String* s = new(mem)String(strlen(str));
memcpy(s->str_, str, s->len_);
// ...
// 当不使用 String 时，在不释放内存的情况下销毁 String 对象
s->~String();  // 释放
