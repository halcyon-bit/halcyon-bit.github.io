列表初始化(C++11)

C++11
后端
发布日期:   2020-10-13
文章字数:   3.1k
阅读时长:   13 分

一、C++98/03 初始化

// 普通数组
int arr[3] = { 1, 2, 3 };

// POD(plain old data) 用来表明C++中与C相兼容的数据类型
struct TestA
{
    int x;
    struct TestB
    {
        int i;
        int j;
    }b;
}a = { 1, {2, 3} };

// 拷贝初始化
int i = 0;
class Test
{
public:
    Test(int) {};
}T = 123;

// 直接初始化
int j(0);

这些不同的初始化方法都有各自的适用范围，但是种类繁多的方法却没有一种可以通用的，所以 C++11 提出了初始化列表的方法来解决此问题。

二、快速初始化成员变量

在 C++98/03 中，支持在类声明中使用等号”=”加初始值的方式，来初始化类中静态成员常量。如果静态成员不满足常量性，则不可以就地初始化，而且即使静态成员常量也只能是整型或者枚举才能就地初始化，而非静态成员变量的初始化则必须在构造函数中进行。

class Test
{
public:
    Test() : a_(0) {}
    Test(int d) : a_(d) {}

private:
    int a_;
    const static int b_ = 0;
    int c_ = 1;  // error, 无法编译通过
    static int d_ = 0;  // 无法编译通过
    static const double e_ = 1.3;  // 非整型或者枚举，无法编译通过
    static const char* const f_ = "e";  // 非整型或者枚举，无法编译通过
};

C++11 中可以使用 {} 就地初始化非静态成员，但是列表初始化的效果总是优于就地初始化的。对于非常量的静态成员变量，还是需要到头文件以外去定义。

#include <iostream>

class TestA
{
public:
    TestA()
    {
        std::cout << "TestA(), num: " << num_ << std::endl;
    }
    TestA(int i) : num_(i)
    {
        std::cout << "TestA(int), num: " << num_ << std::endl;
    }

    int num_{ 2 };
};

class TestB
{
public:
    TestB()
    {
        std::cout << "TestB(), val: " << val_ << std::endl;
    }
    TestB(int i) : val_('G'), a(i)
    {
        std::cout << "TestB(int), val: " << val_ << std::endl;
    }
    void num_of_a()
    {
        std::cout << "number of A: " << a.num_ << std::endl;
    }
    void num_of_b()
    {
        std::cout << "number of B: " << b.num_ << std::endl;
    }

private:
    char val_{ 'g' };
    TestA a;
    TestA b{ 19 };
};

int main()
{
    TestA a;  // TestA(), num: 2
    TestB b;  // TestA(), num: 2
    // TestA(int), num: 19
    // TestB(), val: g
    b.num_of_a();  // number of A: 2
    b.num_of_b();  // number of B: 19

    TestB c(7);  // TestA(int), num: 7
    // TestA(int), num: 19
    // TestB(int), val: G
    c.num_of_a();  // number of A: 7
    c.num_of_b();  // number of B: 19
    return 0;
}

三、统一初始化方法——列表初始化

C++98/03 中已经存在初始化列表的方法，只是范围比较窄，只适用于常规 POD 类型。

int arr1[3] = { 1, 2, 3 };
int arr2[] = { 1, 2, 3, 4 };
struct Test
{
    int i;
    int j;
}t = { 1, 2 };

C++11 将这种初始化方法适用于所有类型的初始化，可以使用以下几种形式完成初始化工作：

  • 等号”=”加上赋值表达式，比如 int a = 3 + 4;
  • 等号”=”加上花括号式的初始化列表，比如 int a = { 3 + 4 };
  • 圆括号式的表达式列表，比如 int a(3+4);
  • 花括号式的初始化列表，比如 int a{ 3+4 };
class Test
{
public:
    Test(int) {};
};

void func()
{
    Test v1(123);
    Test v2 = { 123 };
    Test v3{ 123 };
    int v4 = { 2 };
    int v5{ 3 };
}

新的初始化方法是变量名后面加 {} 来进行初始化，{} 内则是初始化的内容，等号是否存在并不影响(编译器会进行优化)。

type val{};

初始化列表支持 new 操作符，也可以作为函数的返回值：

int* a = new int{ 5 };
int* arr = new int[3] {1, 2, 3};、

std::vector<int> func()
{
    return { 1, 2, 3 };
}

在 C++11 中，初始化列表是非常方便的，不仅统一了对象的初始化方式，还使代码更加简洁清晰。

1. std::initializer_list

在 C++11 中，对于 stl 容器和未显示数组长度的数组可以进行任意长度的初始化。

int arr[] = { 1,2,3,4 };
std::vector<int> vec = { 1,2,3,4 };
std::map<std::string, int> tmap = { {"1", 1}, {"2", 2}, {"3", 3} };

但是对于自定义类型不具备这种能力，但是 C++11 解决了这个问题，C++11 中可以通过轻量级模板 std::initalizer_list 来解决这个问题。只需要添加一个参数为 std::initializer_list 的构造函数，这个自定义类型即可拥有这种任意长度初始化列表来初始化的能力。

class Test
{
public:
    Test(std::initializer_list<int> vals)
    {
        for (auto& each : vals) {
            std::cout << each << std::endl;
        }
    }
};

Test t{ 1,2,3,4,5 };

std::initializer_list 负责接收初始化列表，可以通过 for 循环来读取其中的元素，并对元素做操作。同样的，可以作为函数参数传递同类型的数据集合。在任何需要的时候，都可以使用其来一次性传递多个同类型参数。

#include <map>
#include <iostream>

using pair_t = std::map<int, int>::value_type;
class Test
{
public:
    Test(std::initializer_list<pair_t> list)
    {
        for (auto& each : list) {
            map_.insert(each);
        }
    }

private:
    std::map<int, int> map_;
};
Test t = { {1,2}, {3,4}, {5,6} };

void func(std::initializer_list<int> list)
{
    for (auto& each : list) {
        std::cout << each << std::endl;
    }
}

int main()
{
    func({});
    func({ 1,2,3,4 });
    return 0;
}

2. 使用细节

std::initializer_list 的实现：

template <class _Elem>
class initializer_list
{
public:
    using value_type = _Elem;
    using reference = const _Elem&;
    using const_reference = const _Elem&;
    using size_type = size_t;

    using iterator = const _Elem*;
    using const_iterator = const _Elem*;

    constexpr initializer_list() noexcept : _First(nullptr), _Last(nullptr) {}

    constexpr initializer_list(const _Elem* _First_arg, const _Elem* _Last_arg) noexcept
        : _First(_First_arg), _Last(_Last_arg) {}

    _NODISCARD constexpr const _Elem* begin() const noexcept {
        return _First;
    }

    _NODISCARD constexpr const _Elem* end() const noexcept {
        return _Last;
    }

    _NODISCARD constexpr size_t size() const noexcept {
        return static_cast<size_t>(_Last - _First);
    }

private:
    const _Elem* _First;
    const _Elem* _Last;
};

std::initializer_list 的特点如下：

  • 编译底层使用的是 std::array，std::initializer_list 保存的是 array 的引用；

  • 它是一个轻量级的容器类型；

  • 可以接收任意长度的初始化列表，但是要求元素必须都是同种类型；

  • 有三个成员接口，size()，begin()，end()；

  • 只能被整体初始化或者赋值。

std::initializer_list 的访问只能通过 begin() 和 end() 来进行循环遍历，遍历取得的迭代器是只读的，所以无法修改其中元素的值，但是可以整体赋值来修改。

std::initializer_list<int> ini_list;
size_t len = ini_list.size();  // len = 0
ini_list = { 1,2,3,4,5 };
len = ini_list.size();  // len = 5
ini_list = { 1,2 };
len = ini_list.size();  // len = 2

std::initializer_list 是非常高效的，它的内部并不保存初始化列表元素中的拷贝，仅仅保存初始化列表中的引用。

如果按照下面的代码来使用 std::initializer_list 是错误的，虽然可以正常通过编译，但是可能无法得到我们希望的结果，因为 a，b 在函数结束时生存周期也结束了，返回的是不确定的内容。

std::initializer_list<std::vector<std::string>> func2()
{
    std::vector<std::string> vec1;
    for (int i = 0; i < 100; i++) {
        vec1.push_back(std::to_string(i));
    }
    std::vector<std::string> vec2;
    for (int i = 100; i < 200; i++) {
        vec2.push_back(std::to_string(i));
    }
    return { vec1, vec2 };  // vec1, vec2 在返回时并没有拷贝
}

正确的用法是，通过真正的容器或者具有转移拷贝语意的容器来替代 std::initializer_list。

std::vector<std::vector<std::string>> func2()
{
    std::vector<std::string> vec1;
    for (int i = 0; i < 100; i++) {
        vec1.push_back(std::to_string(i));
    }
    std::vector<std::string> vec2;
    for (int i = 100; i < 200; i++) {
        vec2.push_back(std::to_string(i));
    }
    return { vec1, vec2 };  // vec1, vec2 在返回时并没有拷贝
}

四、防止类型收窄

1. 类型收窄的情况

class Test
{
public:
    Test(int i)
    {
        std::cout << i << std::endl;
    }
};

Test t(1.2);

这个例子就是类型收窄的情况，虽然说能够正常通过编译，但是在传递 i 之后不能完整的保存浮点数的数据。

C++ 中类型收窄的情况：

  • 从一个浮点数隐式转换为一个整数，如 int i = 2.2;
  • 从高精度浮点数隐式转换为低精度浮点数，如 long doule 隐式转换为 double 或者 float；
  • 从一个整数隐式转换为一个浮点数，并且超出了浮点数的范围，如 float f = (unsigned long long)–1；
  • 从一个整型隐式转换为一个长度较短的整型数，并且超出了长度较短的整型数范围，如 char x = 65536。

这些类型收窄的情况，在编译器并不会报错，但是可能存在潜在的错误。

2. C++11 的改善

C++11 中可以通过初始化列表来检查，防止类型的收窄。

int n1 = 1.1;  // ok
int n2 = { 1.1 };  // error
float f1 = 1e40;  // ok
float f2 = { 1e40 };  // error
float f3 = (unsigned long long)-1;  // ok
float f4 = { (unsigned long long)-1 };  // error
float f5 = (unsigned long long)1;  // ok
float f6 = { (unsigned long long)1 };  // ok
const int x = 1024, y = 1;
char c1 = x;  // ok
char c2 = { x };  // error
char c3 = y;  // ok
char c4 = { y };  // ok

五、聚合类型

1. 自定义类型初始化

在 C++11 中使用初始化列表时，可能有以下情况：

struct TestA
{
    int x;
    int y;
}a = { 123,123 };  // a.x = 123, a.y = 123

struct TestB
{
    int x;
    int y;
    TestB(int, int) : x(0), y(0) {}
}b = { 123,123 };  // b.x = 0, b.y = 0

TestA 是以 C++98/03 的聚合类型来初始化的，用拷贝的方式初始化 TestA 中的成员，而 TestB 由于自定义了构造函数，所以初始化是以构造函数来初始化的。

当使用初始化列表时，如果是聚合类型，则以拷贝的方式来初始化成员，如果是非聚合类型，则是以构造函数来初始化成员。

2. 聚合类型

  1. 类型是普通数组（int[10]，char[]，long[2][3]等）；
  2. 类型是类，且：
    • 无用户自定义构造函数；
    • 无私有或者保护的非静态成员；
    • 无基类；
    • 无虚函数；
    • 无{}和=直接初始化的非静态数据成员。

2.1 数组

对于数组而言，只要该类型是一个普通的数组，如果数组的元素并不是聚合类型，那么这个数组仍是一个聚合类型。

int a[] = { 1, 2, 3 };
std::string strarr[3] = { "hello", "C++", "11" };

2.2 存在自定义构造函数

struct Test
{
    int x;
    int y;
    int z;

    Test(int, int) {};
};
Test a = { 123, 123, 12 };  // error

当一个自定义类或结构体拥有自己的构造函数，无法将该类看作一个聚合类型，必须通过自定义的构造函数才能构造对象。

2.3 存在私有或者保护的非静态数据成员

struct TestA
{
    int x;
    int y;
protected:
    int z;
};

TestA a = { 123, 123, 12 };   // error

struct TestB
{
    int x;
    int y;
protected:
    static int z;
};
TestB b = { 123, 123 };       // ok

TestA 的实例化是失败的，因为 z 是一个受保护的非静态成员。而 TestB 是成功的，因为 z 是一个受保护的静态数据成员，所以，类成员里面的静态数据成员是不能通过初始化列表来初始化的，静态数据成员的初始化遵循静态成员的初始化方式。

2.4 基类或者虚函数

有基类或者虚函数同样不适用于使用初始化列表。

struct TestA
{
    int x;
    int y;
    virtual void fun() {};
};
TestA a = { 123, 123 };       // error

struct Base
{};

struct Derived : public Base
{
    int x;
    int y;
};
Derived b = { 123, 123 };       // error

2.5 {} 和 = 初始化的非静态数据成员

struct Test
{
    int x;
    int y = 2;
};

Test a = { 123, 123 };  // error

y 在声明时被初始化为 2，所以 A 不再是一个聚合类型。

这个例子中需要注意的是，C++11 中放宽了类型声明的初始化操作，即在非静态数据成员的声明时调用 {} 或者 = 来对成员进行初始化，但是造成的影响是该类型不再是聚合类型，所以不能直接使用初始化列表。如果要使用初始化列表就必须自己定义一个构造函数。

3. 聚合类型并非递归

struct TestA
{
    int x;
    int y;
private:
    int z;
};
TestA a{ 1, 2, 3 };   // error

TestA a1{};         // ok

struct TestB
{
    TestA a;
    int x;
    double y;
};

TestB b{ {}, 1, 2.5 };

TestA 有一个私有化的非静态成员，所以使用 a 是错误的，但是可以调用无参构造函数，所以在 TestB 中，即使成员 a 是一个非聚合类型，但是 TestB 仍然是一个聚合类型，可以直接使用初始化列表。

4. 小结

对于一个聚合类型，使用初始化列表相当于对其中每个元素分别赋值；而对于非集合类型，则需要先定义一个合适的构造函数，此时使用初始化列表将调用对应的构造函数。
