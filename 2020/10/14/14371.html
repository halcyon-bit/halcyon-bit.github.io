智能指针(C++11)

C++11
后端
发布日期:   2020-10-14
文章字数:   3.8k
阅读时长:   17 分

C++11 引入了三种智能指针：

  • std::shared_ptr
  • std::weak_ptr
  • std::unique_ptr

一、std::shared_ptr

std::shared_ptr 使用引用计数，每个 std::shared_ptr 的拷贝都指向相同的内存，在最后一个 std::shared_ptr 析构的时候，内存才会释放。

1. 初始化

  • 通过 std::make_shared<T>()；
  • 直接构造 std::shared_ptr；
  • 调用 reset() 方法初始化。

智能指针的用法和普通指针类似，不过不需要自己管理分配的内存，对于没有初始化的智能指针，只能通过 reset() 来初始化；当智能指针有值时，reset() 会使计数器减 1。

注意：智能指针不能通过原始指针来初始化。

优先使用 std::make_shared 来构造智能指针，更高效。

#include <iostream>
#include <memory>

int main()
{
    // 1. 使用 std::make_shared<T>()
    std::shared_ptr<int> p1 = std::make_shared<int>(10);
    // 2. 直接构造
    std::shared_ptr<int> p2(new int(20));
    std::shared_ptr<int> p3 = p2;
    std::shared_ptr<int> p4;
    // 3. 使用 reset() 方法
    p4.reset(new int(30));
    if (p4) {
        std::cout << "p4:" << *p4 << std::endl;
    }

    // error: 无法使用原始指针直接初始化
    std::shared_ptr<int> p = new int(1); // 编译报错，不能直接赋值
    return 0;
}

2. 成员方法

T* get() const noexcept;

当需要获取原始指针的时候，可以通过 get() 方法来获取原始指针。注意：不能释放，如果释放会出错。

long use_count() const noexcept;

该方法获得当前资源的引用计数。

std::shared_ptr<int> ptr(new int(1));
int* p = ptr.get();
delete p; // 会导致 ptr 释放时异常，即二次释放

std::shared_ptr<int> p1(new int(1));
std::shared_ptr<int> p2 = p1;
std::shared_ptr<int> p3 = p2;
p3.use_count();  // 3

3. 指定删除器

智能指针支持指定删除器，即在指针引用为 0 的时候自动调用，支持普通函数和 lambda 表达式。

#include <memory>

void delete_ptr(int* p)
{
    delete p;
}

int main()
{
    std::shared_ptr<int> p1(new int(10), delete_ptr);
    std::shared_ptr<int> p2(new int(20), [](int* p) { delete p; });
    return 0;
}

当智能指针管理动态数组的时候，需要指定删除器，默认的删除器不支持数组对象。自定义删除器或者使用改善的默认修改器都可以。

std::shared_ptr<int> p1(new int[10], [](int* p) {delete[] p; });  // lambda
// std::default_delete 为默认删除器，内部通过调用 delete 实现。
std::shared_ptr<int> p2(new int[20], std::default_delete<int[]>());  // 指定 delete[]

template<typename T>
std::shared_ptr<T> make_shared_array(size_t size)
{
    return std::shared_ptr<T>(new T[size], std::default_delete<T[]>());
}

4. 注意

  • 避免用一个原始指针初始化多个 std::shared_ptr，否则会出现 double free，导致程序崩溃；
int* p = new int;
std::shared_ptr<int> p1(p);
std::shared_ptr<int> p2(p);
  • 通过 shared_from_this() 返回 this 指针。不要将 this 指针作为 std::shared_ptr 返回出来，因为 this 指针本质是一个裸指针，这样可能导致重复析构。
#include <memory>
#include <iostream>

class TestA
{
public:
    std::shared_ptr<TestA> get_self()
    {
        return std::shared_ptr<TestA>(this);
    }

    ~TestA()
    {
        std::cout << "TestA is deleted." << std::endl;
    }
};

int main()
{
    std::shared_ptr<TestA> p1(new TestA);
    std::shared_ptr<TestA> p2 = p1->get_self();
    return 0;
}
// 输出
// TestA is deleted.
// TestA is deleted.
// 崩溃

解决办法：让目标类派生自 std::enable_shared_from_this<T> 类，然后使用基类的成员函数 shared_from_this 来返回 this 的 std::shared_ptr。

class TestA : public std::enable_shared_from_this<TestA>
{
public:
    std::shared_ptr<TestA> get_self()
    {
        return shared_from_this();
    }

    ~TestA()
    {
        std::cout << "TestA is deleted." << std::endl;
    }
};
  • 避免循环使用，循环使用可能导致内存泄漏
#include <memory>
#include <iostream>

class TestB;
class TestA
{
public:
    std::shared_ptr<TestB> bptr_;

    ~TestA()
    {
        std::cout << "TestA is deleted." << std::endl;
    }
};

class TestB
{
public:
    std::shared_ptr<TestA> aptr_;

    ~TestB()
    {
        std::cout << "TestB is deleted." << std::endl;
    }
};

int main()
{
    std::shared_ptr<TestA> pa(new TestA);
    std::shared_ptr<TestB> pb(new TestB);

    pa->bptr_ = pb;
    pb->aptr_ = pa;
    return 0;
}

程序结果是两个指针 pa 和 pb 都不会删除，存在内存泄漏。循环引用导致 pa 和 pb 的引用计数为 2，离开作用域之后，pa 和 pb 的引用计数为 1，并不会减为 0，导致两个指针都不会析构而产生内存泄漏，解决方法是使用 std::weak_ptr。

  • 不要在参数中创建 std::shared_ptr，不同的编译器可能有不同的调用约定，如果先 new int，然后调用 func1，在 func1 过程中发生异常，但是 std::shared_ptr 没有创建，那么会导致内存泄漏。应该先创建智能指针。
func(std::shared_ptr<int>(new int(1)), func1());

std::shared_ptr<int> p(new int(1));
func(p, func1());
  • 尽量使用 std::make_shared，少用 new；
  • 不要删除 get() 返回的原始指针；
  • 不是 new 出来的空间要自定义删除器。

二、std::unique_ptr

std::unique_ptr 是一个独占型智能指针，它不允许其他的智能指针共享其内部的指针，不允许复制，支持移动，通过 std::move() 来转移到其他的 std::unique_ptr，但是转移之后，它本身不再拥有原来指针的所有权。

1. 初始化

std::unique_ptr 没有 std::make_unique() 辅助函数来创建对象。

std::unique_ptr<int> p1(new int(10));
std::unique_ptr<int> p2 = p1;  // error
std::unique_ptr<int> p3 = std::move(p1);  // p1 将变为 nullptr
std::unique_ptr<int> p4;
p4.reset(new int(20));

2. release 方法

pointer release() noexcept;

该方法释放管理对象的所有权，并返回指向该对象的指针。调用 release 后，std::unique_ptr 将不再管理该对象，get() 方法将返回 nullptr，调用者需要负责释放该对象。

3. 数组

std::unique_ptr 除了独占之外，还可以指向数组。

std::unique_ptr<int[]> p1(new int[10]);  // ok
p1[1] = 10;

std::shared_ptr<int[]> p1(new int[10]); // error

4. 指定删除器

std::unique_ptr 必须指定删除器类型。

std::shared_ptr<int> p1(new int(1), [](int* p) { delete p; });  // ok
std::unique_ptr<int> p2(new int(1), [](int* p) { delete p; });  // error
std::unique_ptr<int, void(*)(int*)> p3(new int(1), [](int* p) { delete p; });  // ok

指定函数类型，然后通过 lambda 表达式是可以，但是如果捕获了变量将会编译报错，因为 lambda 表达式在没有捕获变量的情况下可以直接转换为函数指针，但是捕获了变量就无法转换。如果要支持，可以通过 std::function 来解决。

std::unique_ptr<int, void(*)(int*)> p1(new int(1), [&](int* p) { delete p; });  // error
std::unique_ptr<int, std::function<void(int*)>> p2(new int(1), [&](int* p) { delete p; });  // ok

同样也支持自定义删除器。

#include <memory>
#include <iostream>

class DeletePtr
{
public:
    void operator()(int* p)
    {
        std::cout << "delete" << std::endl;
        delete p;
    }
};

int main()
{
    std::unique_ptr<int, DeletePtr> p(new int(1));
    return 0;
}

三、std::weak_ptr

弱引用智能指针 std::weak_ptr 用来监视 std::shared_ptr，其不会使引用计数加 1，也不管理 std::shared_ptr 内部的指针，主要是监视 std::shared_ptr 的生命周期。std::weak_ptr 不共享指针，不能操作资源，它的构造和析构都不会改变引用计数。

使用 std::shared_ptr 来初始化 std::weak_ptr。

1. 观测引用计数

long use_count() const noexcept;

该方法获得当前资源的引用计数。

std::shared_ptr<int> sp(new int(10));
std::weak_ptr<int> wp(sp);
std::cout << wp.use_count() << std::endl; // 输出 1
std::shared_ptr<int> sp1(sp);
wp.use_count();  // 2

2. 监视 std::shared_ptr 是否有效

bool expired() const noexcept;

该方法判断所观察的资源是否已被释放，返回 true 表示指针已失效。

std::shared_ptr<int> sp(new int(10));
std::weak_ptr<int> wp(sp);

wp.expired();  // false, 指针有效
sp.reset();
wp.expired();  // true，指针失效

3. 获取

std::shared_ptr<T> lock() const noexcept;

该方法获取所监视的 std::shared_ptr，如果当前 std::weak_ptr 已经过期，则该函数会返回一个空的 std::shared_ptr 指针。

⭐注意：lock() 方法是原子操作。

#include <memory>
#include <iostream>

std::weak_ptr<int> wptr;
void func()
{
    // 监听是否释放
    if (wptr.expired()) {
        std::cout << "ptr 已经释放" << std::endl;
    }
    else {
        auto sp = wptr.lock();
        std::cout << *sp << std::endl;
    }
}

int main()
{
    {
        auto sptr = std::make_shared<int>(20);
        wptr = sptr;
        func();  // 输出：20
    }
    func();  // 输出：ptr 已经释放
    return 0;
}

4. 返回 this 指针

std::shared_ptr 不能直接返回 this 指针，需要通过派生 std::enable_shared_from_this 类，并通过其方法 shared_from_this() 来返回智能指针，因为 std::enable_shared_from_this 类中有一个 std::weak_ptr，这个 std::weak_ptr 用来观测 this 指针，调用 shared_from_this() 方法时，调用了内部的 std::weak_ptr 的 lock() 方法，将所观测的 std::sharerd_ptr 返回。

#include <memory>
#include <iostream>

class TestA : public std::enable_shared_from_this<TestA>
{
public:
    std::shared_ptr<TestA> get_self()
    {
        return shared_from_this();
    }

    ~TestA()
    {
        std::cout << "TestA is deleted." << std::endl;
    }
};

int main()
{
    std::shared_ptr<TestA> p1(new TestA);
    std::shared_ptr<TestA> p2 = p1->get_self();  //ok
    return 0;
}
// 输出： 
// TestA is deleted.

5. 解决循环引用问题

std::shared_ptr 的循环引用可能导致内存泄漏，通过 std::weak_ptr 可以解决这个问题，将 TestA 或者 TestB 任意一个成员变量改为 std::weak_ptr 即可。

#include <memory>
#include <iostream>

class TestB;
class TestA
{
public:
    std::shared_ptr<TestB> bptr_;

    ~TestA()
    {
        std::cout << "TestA is deleted." << std::endl;
    }
};

class TestB
{
public:
    std::weak_ptr<TestA> aptr_;

    ~TestB()
    {
        std::cout << "TestB is deleted." << std::endl;
    }
};

int main()
{
    std::shared_ptr<TestA> pa(new TestA);
    std::shared_ptr<TestB> pb(new TestB);

    pa->bptr_ = pb;
    pb->aptr_ = pa;
    return 0;
}

// 输出：
// TestA is deleted.
// TestB is deleted.

四、标准库中 std::shared_ptr 的实现

std::shared_ptr 内部是使用引用计数来记录托管指针被引用的次数，当托管指针的引用计数为 0 时，会释放托管的内存，通过 gcc 源码探究 std::shared_ptr 内部究竟是如何实现内存引用计数功能。

类图如下：

shared_ptr 类几乎什么都没有做，继承 __shared_ptr，其内部有一个类型为 __shared_count 的成员，__shared_count 类内部有类型为 _Sp_counted_base 的 _M_pi 的成员，_Sp_counted_base 才是整个 shared_ptr 功能的核心，通过 _Sp_counted_base 控制引用计数来管理内存，由图可见 _Sp_counted_base 内部不持有托管内存的指针。

__shared_count 内部的 _M_pi 成员其实是一个继承自 _Sp_counted_base 的 _Sp_counted_ptr 类型，_Sp_counted_ptr 类型内部持有托管内存的指针 _M_ptr，_M_pi 是一个 _Sp_counted_base 基类对象指针，指向 _Sp_counted_ptr 子类对象内存，这样 _M_pi 内部就既可以控制引用计数，又可以在最后释放托管内存。

这里称 _M_pi 为管理对象，其内部的 _M_ptr 为托管对象。管理对象内部有两个成员变量 _M_use_count 和 _M_weak_count。_M_use_count 表示托管对象的引用计数，控制托管对象什么时候析构和释放。假如有 N 个 shared_ptr 的拷贝，那引用计数就是 N，当引用计数为 0 时调用托管对象的析构函数并释放内存。_M_weak_count 表示管理对象的引用计数，管理对象也是一个内存指针，该指针是创建第一个 shared_ptr 时 new 出来的，到最后也需要释放，所以使用 _M_weak_count 来控制管理对象什么时候析构，平时用到的 weak_ptr 内部其实持有的就是这个管理对象的指针，当 weak_ptr 拷贝时，管理对象的引用计数 _M_weak_count 就会增加，当 _M_weak_count 为 0 时，管理对象 _M_pi 就会析构且释放内存。

1. _M_use_count 的加减

_M_use_count 表示托管对象的引用计数，即当 shared_ptr 拷贝时增加，当 shared_ptr 析构时减少：

// shared_ptr 的拷贝构造函数
template<typename _Yp>
__shared_ptr(const __shared_ptr<_Yp, _Lp>& __r, element_type* __p) noexcept
    : _M_ptr(__p), _M_refcount(__r._M_refcount) // never throws
{}

template<typename _Yp, typename = _Compatible<_Yp>>
__shared_ptr(const __shared_ptr<_Yp, _Lp>& __r) noexcept
    : _M_ptr(__r._M_ptr), _M_refcount(__r._M_refcount)
{}

// __shared_count 的拷贝构造函数
__shared_count(const __shared_count& __r) noexcept
    : _M_pi(__r._M_pi)
{
    if (_M_pi != nullptr)
        _M_pi->_M_add_ref_copy();  // 使引用计数加 1
}

// 增加引用计数
template<>
inline void _Sp_counted_base<_S_single>::_M_add_ref_copy()
{
    ++_M_use_count;
}

shared_ptr 的赋值操作符：

// shared_ptr 的赋值操作符
template<typename _Yp>
_Assignable<const shared_ptr<_Yp>&> operator=(const shared_ptr<_Yp>& __r) noexcept
{
    this->__shared_ptr<_Tp>::operator=(__r);
    return *this;
}

// __shared_ptr 的赋值操作符
template<typename _Yp>
_Assignable<_Yp> operator=(const __shared_ptr<_Yp, _Lp>& __r) noexcept
{
    _M_ptr = __r._M_ptr;
    _M_refcount = __r._M_refcount; // __shared_count::op= doesn't throw
    return *this;
}

// __shared_count 的赋值操作符
__shared_count& operator=(const __shared_count& __r) noexcept
{
    _Sp_counted_base<_Lp>* __tmp = __r._M_pi;
    if (__tmp != _M_pi)
    {
        if (__tmp != nullptr)
            __tmp->_M_add_ref_copy();  // 增加新托管对象的引用计数
        if (_M_pi != nullptr)
            _M_pi->_M_release();  // 减少原有托管对象的引用计数
        _M_pi = __tmp;
    }
    return *this;
}

shared_ptr 析构时会调用 _shared_count 的析构函数，进而减少 _M_use_count 的计数，当减为 0 时，释放内存：

~__shared_ptr() = default;

~__shared_count() noexcept
{
    if (_M_pi != nullptr)
        _M_pi->_M_release();  // 减少引用计数
}

template<>
inline void _Sp_counted_base<_S_single>::_M_release() noexcept
{
    if (--_M_use_count == 0)
    {
        _M_dispose();  // 为0，释放托管对象
        if (--_M_weak_count == 0)
            _M_destroy();  // 为0，释放管理对象
    }
}

virtual void _Sp_counted_ptr::_M_dispose() noexcept
{
    delete _M_ptr;  // 释放托管对象
}

在 shared_ptr 生命周期结束，析构时会将引用计数减 1，如果引用计数为 0，会调用 _M_dispose() 函数进而释放托管对象内存。

2. _M_weak_count 的加减

上面的代码中可以看出 --_M_weak_count == 0 时，会调用 _M_destroy() 函数。

管理对象初始化时 _M_weak_count 的初始值为 1。

_Sp_counted_base() noexcept 
    : _M_use_count(1), _M_weak_count(1)
{}

注意：当 shared_ptr 拷贝或者移动时， _M_weak_count 是不会增加的，它表示的是管理对象的计数，只有当 _M_use_count 为 0 时， _M_weak_count 才会减 1，除此之外 _M_weak_count 的数值是由 weak_ptr 控制的。

从类图可以看出 weak_ptr 内部其实和 shared_ptr 内部持有的是同一个管理对象指针，即 _Sp_counted_base 的指针，当 weak_ptr 拷贝、析构时，_Sp_counted_base 内部的 _M_weak_count 会相应加减。

// 拷贝构造
__weak_count(const __shared_count<_Lp>& __r) noexcept
    : _M_pi(__r._M_pi)
{
    if (_M_pi != nullptr)
        _M_pi->_M_weak_add_ref();
}

template<>
inline void _Sp_counted_base<_S_single>::_M_weak_add_ref() noexcept
{
    ++_M_weak_count;  // 增加管理对象引用计数
}

// 析构函数
~__weak_count() noexcept
{
    if (_M_pi != nullptr)
        _M_pi->_M_weak_release();  // 减少管理对象引用计数
}

template<>
inline void _Sp_counted_base<_S_single>::_M_weak_release() noexcept
{
    if (--_M_weak_count == 0)
        _M_destroy();  // 为0，释放管理对象
}

virtual void _Sp_counted_base::_M_destroy() noexcept
{
    delete this;
}

weak_ptr 拷贝时 _M_weak_count 加 1，析构时 _M_weak_count 减 1，当 _M_weak_count 为 0 时，表示不再需要管理对象来控制托管对象，调用 _M_destroy() 的 delete this 来释放管理对象内存。

3. weak_ptr 的 expired() 和 lock()

bool __weak_ptr::expired() const noexcept
{
    return _M_refcount._M_get_use_count() == 0;
}

// 标准规定该操作为原子操作
__shared_ptr<_Tp, _Lp> __weak_ptr::lock() const noexcept
{
    return __shared_ptr<element_type, _Lp>(*this, std::nothrow);
}

__shared_ptr(const __weak_ptr<_Tp, _Lp>& __r, std::nothrow_t) noexcept
    : _M_refcount(__r._M_refcount, std::nothrow)
{
    _M_ptr = _M_refcount._M_get_use_count() ? __r._M_ptr : nullptr;
}

4. shared_from_this()

template<typename _Tp>
class enable_shared_from_this
{
    shared_ptr<_Tp> shared_from_this()
    {
        return shared_ptr<_Tp>(this->_M_weak_this);
    }

    shared_ptr<const _Tp> shared_from_this() const
    {
        return shared_ptr<const _Tp>(this->_M_weak_this);
    }

    mutable weak_ptr<_Tp> _M_weak_this;
};

使用 shared_from_this() 的类需要继承 std::enable_shared_from_this 类，std::enable_shared_from_this 类中持有一个类型为 weak_ptr 的成员 _M_weak_this，调用 shared_from_this() 就是将内部持有的 weak_ptr 转成了 shared_ptr。

5. 总结

  • shared_ptr 内部使用 __shared_count 中的 _Sp_counted_base 对象来控制托管指针；
  • _Sp_counted_base 内部有 _M_use_count 和 _M_weak_count；
    • _M_use_count 表示托管指针的引用计数，_M_use_count 为 0 时，释放托管指针指向的内存；
    • _M_weak_count 表示 _Sp_counted_base 的引用计数，_M_weak_count 为 0 时，释放 _Sp_counted_base 指向的内存；
  • _Sp_counted_base 的生命周期一般不会短于 shared_ptr 的生命周期。
