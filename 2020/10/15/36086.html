左值右值(C++11)

C++ C++11
后端
发布日期:   2020-10-15
文章字数:   3.7k
阅读时长:   15 分

一、左值、右值

在 C++11 中所有的值必属于左值、将亡值、纯右值三者之一。

左值是指表达式结束后依然存在的持久化对象，右值是指表达式结束时就不再存在的临时对象。所有的具名变量或者对象都是左值，而右值不具名。区分左值和右值的方法：看能不能对表达式取地址，如果能，则为左值，否则为右值。

注意：字符串字面量(“hello world”)是左值。

在 C++11 中，右值分为将亡值和纯右值。

纯右值：非引用返回的临时变量、运算表达式产生的临时变量、除字符串字面量之外的字面量和 lambda 表达式等。

将亡值：C++11 新增的、与右值引用相关的表达式，比如，将要被移动的对象、T&& 函数返回值、std::move 返回值和转换为 T&& 的类型的转换函数的返回值等，将亡值可以理解为即将要销毁的值。

class Test
{
public:
    Test(int a) : a_(a)
    {}

private:
    int a_;
};

Test func()
{
    return Test(10);
}

int main()
{
    int i = 0;  // i是左值，0是纯右值
    Test t1 = func();  // t1是左值，func() 的返回值是右值(临时变量)
    auto t2 = std::move(t1);  // t2 是将亡值
}

左值引用：就是对左值进行引用的类型。

int num = 10;
int& num1 = num;
int& num2 = 1;  // error

右值引用：就是对右值进行引用的类型。语法是 T&&，通过’&&’来表示绑定类型为 T 的右值。通过右值引用的声明，该右值又“重获新生”，其生命周期与右值引用类型变量的生命周期一样，只要该变量还活着，该右值临时量将会一直存活下去。

无论是声明一个左值引用还是右值引用，都必须立即进行初始化。因为引用类型本身并不拥有所绑定对象的内存，只是该对象的一个别名。左值引用是具名变量值的别名，而右值引用则是不具名（匿名）变量的别名。

int&& num1 = 0;
int num2 = 1;
int&& num3 = b;  // error: 右值引用是不能绑定任何左值的

Test test1 = func();
const Test& test2 = func();
Test test3 = func();

最后一行代码会产生两种类型的值，一种是左值 test3，一种是函数 func() 返回的临时值，这个临时值在表达式结束后就销毁了，而左值 test3 在表达式结束后仍然存在，这个临时值就是右值。

常量左值引用：在 C++ 中是个“万能”的引用类型，它可以接收左值、右值、常量左值和常量右值。而且在使用右值对其初始化的时候，常量左值引用还可以像右值引用一样将右值的生命期延长。

注意：普通的左值引用不能接受右值。

二、右值引用

T&& 并不一定表示右值，有可能表示左值，也有可能表示右值，称为 universal references（未定的引用类型）。但是 T&& 必须被初始化，如果被右值初始化就是右值；如果被左值初始化就是左值。

#include <iostream>
#include <type_traits>

template<typename T>
void func1(T&& param)  // T 需要推导
{
    if (std::is_lvalue_reference<decltype(param)>::value) {
        std::cout << "lvalue" << std::endl;
    }
    else if (std::is_rvalue_reference<decltype(param)>::value) {
        std::cout << "rvalue" << std::endl;
    }
}

template<typename T>
void func2(const T&& param);  // param 一定为右值引用

int main()
{
    func1(10);  // rvalue，10 是右值
    int num = 10;
    func1(num);  // lvalue，num 是左值
    
    func2(10);  // ok
    func2(num);  // error: 无法将左值绑定到右值引用上
}

⭐注意：只有当发生自动类型推导时（如函数模板的类型自动推导，或 auto 关键字），&& 才是一个 universal references，并且 universal references 仅仅在 T&& 下发生，任何一点的附加条件都会使之失效，而变成一个普通的右值引用（如 func2）。

引用类型 可以引用的值类型 注记
非常量左值 常量左值 非常量右值 常量右值
非常量左值引用 Y N N N 无
常量左值引用 Y Y Y Y 全能类型，可用于拷贝语义
非常量右值引用 N N Y N 用于移动语义，完美转发
常量右值引用 N N Y Y 暂无用途

标准库 <type_traits> 头文件中提供了3个模板类：std::is_reference、std::is_rvalue_reference、std::is_lvalue_reference，可判断一个类型是否是引用类型，以及是左值引用还是右值引用。

对于 T&& 这种变化被称为引用折叠。C++11 中的折叠引用规则如下：

  • 所有的右值引用叠加到右值引用上还是一个右值引用；
  • 所有的其他引用类型之间的叠加都变成左值引用。
#include <string>
#include <vector>
#include <iostream>

void reference(std::string& str)
{
    std::cout << "lvalue" << std::endl;
}

void reference(std::string&& str)
{
    std::cout << "rvalue" << std::endl;
}

void forward_t(std::string&& str)
{
    reference(str);
}

int main()
{
    std::string str1 = "string";  // str1 是一个左值
    reference(str1);  // lvalue
    reference("string");  // rvalue
    forward_t("string");  // lvalue, forward_t 函数接收的是一个右值，但是在转发给 reference 时又变成了左值。
                          // 因为在 forward_t 中调用 reference 时，右值 str 变成了一个命名的对象，编译器会将其当作左值处理。
    
    std::string&& str2 = std::move(str1);  // ok, std::move 可以将左值转移为右值

    std::cout << "str2 = " << str2 << std::endl;  // string
    std::cout << "str1 = " << str1 << std::endl;  // string, 并没有将 str1 的值转移

    const std::string& str3 = str1 + str1;  // ok, 常量左值引用能够延长临时变量的生命周期(字面值)
    // str3 += "Test";  // 非法, const 无法被修改
    std::cout << "str3 = " << str3 << std::endl;  // stringstring

    std::string&& str4 = str1 + str3;  // ok, 右值引用延长临时对象的生命周期
    str4 += "string";  // ok, 非常量引用能够修改临时变量
    std::cout << "str4 = " << str4 << std::endl;  // stringstringstringstring

    std::vector<std::string> vec;
    vec.push_back(std::move(str1));  // str1 的值转移, str2 的值也没有了
    std::cout << str1 << ' ' << str2 << std::endl;  // ' '
}

&& 的总结如下：

  • 左值和右值是独立于它们的类型的，右值引用类型可能是左值也可能是右值；
  • auto&& 或函数参数类型自动推导的 T&& 是一个未定的引用类型，被称为 universal references，它可能是左值引用也可能是右值引用类型，取决于初始化的值类型；
  • 所有的右值引用叠加到右值引用上仍然是一个右值引用，其他引用折叠都为左值引用。当 T&& 为模板参数时，输入左值，它会变成左值引用，而输入右值时则变为具名的右值引用；
  • 编译器会将已命名的右值引用视为左值，而将未命名的右值引用视为右值。

三、优化性能

引入右值解决两个问题，第一个问题就是临时对象非必要的昂贵的拷贝操作，第二个问题是在模板函数中如何按照参数的实际类型进行转发。

C++ 中引入右值引用和移动语义，可以避免无谓的复制，提高程序性能。

#include <iostream>

class Test
{
public:
    Test()
    {
        std::cout << "construct:" << ++s_construct_count << std::endl;
    }

    ~Test()
    {
        std::cout << "destruct:" << ++s_destruct_count << std::endl;
    }

    Test(const Test& rhs)
    {
        std::cout << "copy construct:" << ++s_copy_count << std::endl;
    }

    static uint32_t s_construct_count;
    static uint32_t s_destruct_count;
    static uint32_t s_copy_count;
};

uint32_t Test::s_construct_count = 0;
uint32_t Test::s_destruct_count = 0;
uint32_t Test::s_copy_count = 0;

Test func()
{
    return Test();
}

int main()
{
    Test t = func();
    return 0;
}

从结果中可以看到，拷贝构造函数调用了两次，一次是 func() 函数内部创建的对象返回出来构造一个临时对象产生的，另一次是在 main 函数中构造 t 对象产生的。第二次的 destruct 是因为临时对象在构造 t 对象之后就销毁了。

Test&& t = func();

func() 返回的右值本应该在表达式语句结束后，其生命也就结束了（因为是临时变量），而通过右值引用，该右值又重获新生，其生命期将与右值引用类型变量 t 的生命期一样，只要 t 还活着，该右值临时变量将会一直存活下去。实际上就是给那个临时变量取了个名字。

通过右值引用，比之前少了一次拷贝构造和一次析构，原因在于右值引用绑定了右值，让临时右值的生命周期延长了。可以利用这个特点做一些性能优化，即避免临时对象的拷贝构造和析构，事实上，在 C++98/03 中，通过常量左值引用也经常用来做性能优化。

const Test& t = func();

输出结果和右值引用一样。

四、移动语义⭐

C++11 中可以将左值强制转换为右值，从而避免对象的拷贝来提升性能。move 将对象的状态或者所有权从一个对象转移到另一个对象，没有内存拷贝。深拷贝和 move 的区别如图：

从图可以看出，深拷贝会有两份内存，而 move 只有一份，move 只是将内存的所有者切换为目标对象，并没有移动任何东西。

在 C++11 之前的拷贝构造函数和赋值函数要有如下定义：

// 赋值函数
T& T::operator=(const T& rhs)
{
    // 销毁内部资源
    // 复制rhs资源到自身
}

T getT(); // 假如 getT 是返回一个 T 类型的函数

T t;
t = getT();

t = getT() 将有如下动作：

  1. 销毁 t 的资源；
  2. 赋值 getT 返回的临时对象的资源；
  3. 销毁临时对象，释放其资源。

其实可以优化的是，将 t 和临时对象的资源指针做交换，让临时对象去销毁 t 原来拥有的资源，t 拥有临时对象的资源指针。那么赋值操作函数就该这么写：

T& T::operator=(const T& rhs)
{
    // 转移资源的控制权，无需复制
}

1. 移动构造函数和移动赋值操作符

C++11 提供了移动构造函数和移动赋值操作符，用于支持移动语义。

class Test
{
public:
    Test(const Test& rhs)
    {
        // 拷贝 rhs.str 的内容到 this.str 中, 需要深拷贝
        str_ = new char[rhs.size_];
        memcpy(str_, rhs.str_, rhs.size_);
        size_ = rhs.size_;
    }

    // 移动构造函数
    Test(Test&& rhs)
    {
        // 夺取 rhs.str 的资源，并将 rhs.str 置空, 无需深拷贝
        str_ = rhs.str_;
        size_ = rhs.size_;
        rhs.str_ = nullptr;
        rhs.size_ = 0;
    }

    Test& operator=(const Test& rhs)
    {
        // 释放 str
        // 拷贝 rhs.str 的内容到 this.str 中, 需要深拷贝
        if (str_ != nullptr) {
            delete[] str_;
        }
        str_ = new char[rhs.size_];
        memcpy(str_, rhs.str_, rhs.size_);
        size_ = rhs.size_;
        return *this;
    }

    // 移动赋值操作符
    Test& operator=(Test&& rhs)
    {
        // 释放 str
        // 夺取 rhs.str 的资源，并将 rhs.str 置空, 无需深拷贝
        if (str_ != nullptr) {
            delete[] str_;
        }
        str_ = rhs.str_;
        size_ = rhs.size_;
        rhs.str_ = nullptr;
        rhs.size_ = 0;
        return *this;
    }

private:
    char* str_{ nullptr };
    size_t size_{ 0 };
};

假如一个临时容器很大，要赋值给另一个容器。如果不用 move，拷贝的代价很大，性能较低，使用 move 几乎没有代价，只转换了资源的所有权。当然，当一个左值被 move 之后，不再对之前的内存拥有控制权，会被置为一个空对象。

注意：在提供移动构造函数的同时也会提供一个拷贝构造函数，以防止移动不成功的时候还能拷贝构造，使代码更安全。

2. std::move

在 C++11 中，标准库 <utility> 中提供了 std::move，该函数并不能移动任何东西，它唯一的功能是将一个左值强制转化为右值引用，继而可以通过右值引用使用该值，以用于移动语义。从实现上讲，std::move 基本等同于一个类型转换：

static_cast<T&&>(lvalue);

注意：被转化的左值，其生命期并没有随着左右值的转化而改变。

使用移动语义的前提是：类需要添加一个以右值引用为参数的移动构造函数和移动赋值操作符，若没有声明该函数，将调用以常量左值引用为参数的拷贝构造函数。这是一种非常安全的设计：移动不成，至少还可以拷贝。

标准库 <type_traits> 头文件中提供了模板类，来判断一个类型是否是可以移动。如：std::is_move_constructible、std::is_trivially_move_constructible、std::is_nothrow_move_constructible。

3. NRVO

编译器一般会对返回值进行优化(NRVO)，但是在没有返回值优化的情况下，编译器将试图把本地对象移动出去，而不是拷贝出去。而使用 std::move 对于移动行为没有帮助，反而会影响返回值优化。

#include <iostream>

class Test
{
public:
    Test()
    {
        std::cout << "Test()" << std::endl;
    }

    Test(const Test&)
    {
        std::cout << "Test(const Test&)" << std::endl;
    }

    Test(Test&&) noexcept
    {
        std::cout << "Test(Test&&)" << std::endl;
    }
};

Test test1()
{
    Test t;
    // 一般会 NRVO
    return t;
}

Test test2()
{
    Test t;
    return std::move(t);
}

Test test3(int n)
{
    Test t1;
    Test t2;
    // 有分支，一般无 NRVO
    if (n % 2 == 0) {
        return t1;
    } else {
        return t2;
    }
}

int main()
{
    std::cout << "**** test1 ****\n";
    Test t1 = test1();
    std::cout << "\n**** test2 ****\n";
    Test t2 = test2();
    std::cout << "\n**** test3 ****\n";
    Test t3 = test3(100);
    return 1;
}

五、完美转发⭐

所谓完美转发，是指在函数模板中，完全依照模板的参数的类型（即保持参数的左值、右值特性），将参数传递给函数模板中调用的另外一个函数。

#include <iostream>
void test(int&& i)
{
    std::cout << "rvalue" << std::endl;
}
void test(int& i)
{
    std::cout << "lvalue" << std::endl;
}
void forward_t(int&& i)
{
    test(i);
}
int main()
{
    forward_t(5);  // lvalue，5 在此时是右值，在 test 调用的时候就是左值
}

C++11 提供了 std::forward，将参数传递给函数中调用的另一个函数，按照参数本来的类型转发。

#include <iostream>
void test(int&& i)
{
    std::cout << "rvalue ref\n";
}
void test(int& i)
{
    std::cout << "lvalue ref\n";
}
void test(const int&& i)
{
    std::cout << "const rvalue ref\n";
}
void test(const int& i)
{
    std::cout << "const lvalue ref\n";
}

template<typename T>
void forward_t(T&& t)
{
    test(std::forward<T>(t));
}

int main()
{
    int a{ 1 }, b{ 0 };
    const int c{ 1 }, d{ 0 };
    forward_t(a);  // lvalue ref
    forward_t(std::move(b));  // rvalue ref
    forward_t(c);  // const lvalue ref
    forward_t(std::move(d));  // const rvalue ref
}
