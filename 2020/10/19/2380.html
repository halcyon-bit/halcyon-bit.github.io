模板特性(C++11)

C++11
后端
发布日期:   2020-10-19
文章字数:   1.8k
阅读时长:   8 分

一、右尖括号

C++98/03 标准：

std::vector<std::list<int> > vec;  // 需要这样写
std::vector<std::list<int>> vec;  // error, >> 会被认为是右移操作符

C++11 标准中，要求编译器对模板的右尖括号做单独处理，使编译器能够正确判断出”>>”是一个右移操作符还是模板参数表的结束标记。

std::vector<std::list<int>> vec;  // ok

二、模板别名

C++11 中出现了可以重定义一个模板的语法：

template <typename T>
using str_map_t = std::map<std::string, T>;

str_map_t<int> map1;

这里使用了 C++11 的关键字 using，使用宏或 typedef 无法达到同样的效果：

#define Vec<T> template<typename T> std::vector<T>;
Vec<int> coll;
// 等同于
template<int> std::vector<int> coll;  // 并不是我们想要的结果

typedef std::vector<int> Vec;  // 类型固定成 int，typedef 不接受参数

using 几乎涵盖了 typedef 的所有功能。总的来说，typedef 有时候阅读难度比 using 更大（比如在定义函数指针的时候），using 别名语法比 typedef 更加清晰。

typedef unsigned int uint_t;
using uint_t = unsigned int;

typedef std::map<std::string, int> map_str_t;
using map_str_t = std::map<std::string, int>;

// 定义函数指针
typedef void (*func_t)(int, int);
using func_t = void(*)(int, int);

// C++98/03
template <typename T>
struct func_t {
    typedef void (*type)(T, T);
};
// 使用 func_t 模板
func_t<int>::type xx_1;

// C++11 
template <typename T>
using func_t = void(*)(T, T);
// 使用 func_t 模板
func_t<int> xx_2;

三、函数模板的默认模板参数

1. 默认模板参数

C++98/03 中，类模版可以有默认模板参数，但是函数模板不支持默认参数。

template <typename T, typename U = int, U N = 0>
class Test
{};

template <typename T = int>     // error in C++98/03: default template arguments
void func(T t)
{}

在 C++11 中解除了这个限制，上面的函数可以直接调用，就像调用正常函数一样：

int main()
{
    int n = 10;
    func(n);
    return 0;
}

当所有模板参数都有默认参数时，函数模板的调用如同一个普通函数。但是对于类模板而言，哪怕所有参数都有默认参数，在使用时也必须在模板名后跟随”<>”来实例化。

2. 细节

函数模板的默认模板参数在使用规则上和其他的默认参数也有些不同，它没有必须写在参数表最后的限制。而类模板的默认模板参数必须从右往左定义。

template <typename T, typename U = int>
class Test
{
    T value;
};

template <typename T = int, typename U> // error
class Test
{
    T value;
};

template <typename T = int, typename U>
U func(T val)
{}

template <typename T, typename U = int>
U func(T val)
{}

模版默认参数可以和自动推导结合使用来提升灵活性。可以在平时的编码中，指定一部分模板参数，另外的一部分自动推导，但是有两条规则：

  • 如果编译器无法自动推导函数模板参数类型，则使用默认模板参数
  • 如果能推导函数模板参数，则使用推导类型
template <typename T>
struct identity
{
    typedef T type;
};

template <typename T = int>
void func1(typename identity<T>::type val, T = 0) {}

void func2()
{
    func1(123);      // T -> int
    func1(123, 123.0);   // T -> double
}

template <typename R = int, typename U>
R func3(U val) {}

int main()
{
    func3(123);
    func3<long>(123); // func 返回类型是 long
    return 0;
}

注意：在调用函数模板时，若显示指定模板的参数，参数填充顺序是从左往右的。

通过 identity 外部模板禁用了参数 val 的类型自动推导，但是 func1 指定了模板参数 T 的默认参数类型，所以 func1(123) 使用默认的 int 类型，而在 func1(123,123.0) 中，第二个参数 123.0 为 double 类型，所以 T 优先被推导为 double 类型。

这里要注意的是，不能将默认模板参数当做模板参数自动推导的”建议”，因为模板参数自动推导总是根据实参推导来的，当自动推导生效时，默认模板参数会被直接忽略。

四、type_tratis

C++11 新增的类型萃取（编译期间获取）。

1. std::decay

template<typename T>
struct decay;

对于普通类型来说，std::decay 是移除引用和 cv 符，也可以用于数组，具体的规则：

  • 先移除 T 类型的引用，得到类型 U，U 定义为 std::remove_reference<T>::type；
  • 如果 std::is_array<U>::value 为 true，修改类型 type 为 std::remove_extent<U>::type*；
  • 否则，如果 std::is_function<U>::value 为 true，修改类型 type 为 std::add_pointer<U>::type；
  • 否则，修改类型 type 为 std::remove_cv<U>::type。
using A = std::decay<int>::type;  // int
using B = std::decay<int&>::type;  // int
using C = std::decay<int&&>::type;  // int
using D = std::decay<const int&>::type;  // int
using E = std::decay<int[2]>::type;  // int*
using F = std::decay<int(int)>::type;  // int(*)int

std::decay 对于函数来说就是添加指针。

2. std::conditional

template<bool B, typename T, typename F>
struct conditional;

在编译期根据一个判断式选择两个类型中的一个，和条件表达式的语义类似，类似于一个三元表达式。

using A = std::conditional<true, int, float>::type;  // int
using B = std::conditional<false, int, float>::type;  // float

3. std::result_of

template<class F, class... Args>
class result_of<F(Args...)>;

在编译期获取一个可调用对象的返回类型。

int func(int)
{
    return 0;
}

typedef int(&func_ref)(int);  // function reference
typedef int(*func_ptr)(int);  // function pointer

struct Test
{
    int operator()(int i)
    {
        return i;
    }
};

using A = std::result_of<decltype(func)&(int)>::type;  // int
using B = std::result_of<func_ref(int)>::type;  // int
using C = std::result_of<func_ptr(int)>::type;  // int
using D = std::result_of<Test(int)>::type;  // int

4. std::enable_if

template<bool B, typename T = void>
struct enable_if;

利用 SFINAE（substitution failure is not an error 替换失败并非错误）实现根据条件选择重载函数。std::enable_if 使得函数在判断条件 B 仅为 true 时才有效。

std::enable_if 可以用于返回值、模板定义、模板特化和参数。

// 用于返回值，只接受整型和浮点
template<typename T>
typename std::enable_if<std::is_arithmetic<T>::value, T>::type func1(T t)
{
    return t;
}

// 对入参类型做了限定，即第二个参数类型为 int
template<typename T>
T func2(T t, typename std::enable_if<std::is_integral<T>::value, int>::type = 0)
{
    return t;
}

// 对模板参数做了限定，只能是 int
template<typename T, class = typename std::enable_if<std::is_integral<T>::value>::type>
T func3(T t)
{
    return t;
}

template<typename T, class U = void>
class Test;

// 模板特化时，对模板参数做了限定，模板参数类型只能为浮点
template<typename T>
class Test<T, typename std::enable_if<std::is_floating_point<T>::value>::type>
{};

template<typename T>
typename std::enable_if<std::is_arithmetic<T>::value, std::string>::type toString(T& t)
{
    return std::to_string(t);
}

template<typename T>
typename std::enable_if<std::is_same<T, std::string>::value, std::string>::type toString(T& t)
{
    return t;
}

int main()
{
    func1(1);  // ok
    func1("1");  // error
    
    func2(1, 1); // ok
    func2(1, "1");  // error
    
    func3(1);  // ok
    func3("1"); // error
    
    Test<double> t;  // ok
    Test<int> t;  // error
    return 0;
}
