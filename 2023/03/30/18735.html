基础-枚举与模式匹配⭐(Rust)

Rust
后端
发布日期:   2023-03-30
文章字数:   933
阅读时长:   4 分

一、枚举

枚举允许列举所有可能的值来定义一个类型。

1. 定义与使用枚举

enum IpAddrKind {
    V4,
    V6,
}

let ip4 = IpAddrKind::V4;
let ip6 = IpAddrKind::V6;

fn route(ip_kind: IpAddrKind) {}

枚举的变体都位于标识符的命名空间下，使用两个冒号 :: 进行分隔。

2. 与结构体结合

struct IpAddr {
    kind: IpAddrKind,
    address: String,
}

fn main() {
    let home = IpAddr {
        kind: IpAddrKind::V4,
        address: String::from("127.0.0.1"),
    };

    let loopback = IpAddr {
        kind: IpAddrKind::V6,
        address: String::from("::1"),
    };
}

3. 将数据附加到枚举的变体中

可以在枚举的变体中嵌入任意的数据类型。

enum IpAddr {
    V4(String),
    V6(String),
}

enum IpAddr {
    V4(u8, u8, u8, u8),
    V6(String),
}

fn main() {
    let home = IpAddr::V4(127, 0, 0, 1);
    let loopback = IpAddr::V6(String::from("::1"));
}
  • 不需要额外使用 struct；
  • 每个变体可以拥有不同的类型以及关联的数据量。
enum Message {
    Quit,  // 无任何数据
    Move{x:i32, y:i32},  // 匿名结构体
    Write(String),  // 字符串
    ChangeColor(i32, i32, i32),  // i32
}

fn main() {
    let q = Message::Quit;
    let m = Message::Move{x:12, y:16};
    let w = Message::Write(String::from("hello world"));
    let c = Message::ChangeColor(255, 255, 255);
}

4. 定义方法

使用 impl 关键字。

enum Message {
    Quit,  // 无任何数据
    Move{x:i32, y:i32},  // 匿名结构体
    Write(String),  // 字符串
    ChangeColor(i32, i32, i32),  // i32
}

impl Message {
    fn call(&self) {}
}

fn main() {
    let q = Message::Quit;
    let m = Message::Move{x:12, y:16};
    let w = Message::Write(String::from("hello world"));
    let c = Message::ChangeColor(255, 255, 255);
    m.call();
}

二、Option 枚举

定义于标准库中，表示某个值可能存在（某种类型）或不存在的情况。因为 Rust 没有 Null，所以提供了类似与 Null 概念的枚举：Option<T>。

标准库中的定义为：

enum Option<T> {
    Some(T),
    None,
}

// 使用
let some_number = Some(5);  // 类型：Option<i32>
let some_string = Some("hello world");  // 类型：Option<&str>
let number: Option<i32> = None;  // 需要显示指定类型，None 无法推导出类型

相较于 Null，使用 Option<T> 使用上更加安全。

let a: i32 = 100;
let b: Option<i32> = Some(200);

let c = a + b;  // error: i32 无法与 Option<i32> 相加

// 在其他语言中
// String a = Null;
// String b = a + "world";  // 可以编译通过，只有在运行期间才可能发现该错误

三、match 运算符⭐

1. 基本用法

允许一个值与一系列模式进行匹配，并执行匹配的模式对应的代码，模式可以是字面值、变量名、通配符等。

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter,
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,  // 匹配到对应的值后，执行 => 后的代码
        Coin::Nickel => 5,
        Coin::Dime => {  // 多行代码需要使用 {}
            println!("Dime");
            10
        },
        Coin::Quarter => 25,
    }
}

匹配的分支可以绑定到被匹配对象的部分值，可以从 enum 变体中提取值。

#[derive(Debug)]
enum UsState {
    Alabama,
    Alaska,
}

enum Coin {
    Penny,
    Nickel,
    Dime,
    Quarter(UsState),
}

fn value_in_cents(coin: Coin) -> u8 {
    match coin {
        Coin::Penny => 1,
        Coin::Nickel => 5,
        Coin::Dime => {
            println!("Dime");
            10
        },
        Coin::Quarter(state) => {
            println!("State quarter from {:?}", state);
            25
        },
    }
}

fn main() {
    let ss = Coin::Quarter(UsState::Alaska);
    println!("{}", value_in_cents(ss));
}

2. 匹配 Option<T>

fn test(x: Option<i32>) -> Option<i32> {
    match x {
        None => None,
        Some(i) => Some(i + 1),
    }
}

fn main() {
    let five = Some(5);
    let six = test(five);
    let none = test(None);
}

3. 必须穷举所有的可能

fn test(x: Option<i32>) -> Option<i32> {
    match x {
        Some(i) => Some(i + 1),
    }
}

上面的代码会编译失败，因为没有对 None 进行处理。解决方案为：使用“ _”通配符，替代其余没列出的可能值。通配符必须要放在最后。

let v = 1u8;
match v {
    1 => println!("one"),
    3 => println!("three"),
    5 => println!("five"),
    7 => println!("seven"),
    _ => (),  // 放在最后，无操作
}

四、if let

match 的语法糖，只关心一种匹配而忽略其它匹配的情况。

let v = Some(0u8);
match v {
    Some(3) => println!("three"),
    _ => (),
}

// 等价的 if let
if let Some(3) = v {
    println!("three");
} else {
    println!("other");
}
